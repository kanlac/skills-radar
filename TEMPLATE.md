# 评测文章模板

撰写新评测时，请按照以下目录结构组织内容。

## 目录结构

```
# 标题（用例名称）

> 一句话描述

本文原载于微信公众号：[文章标题](链接)  <!-- 可选 -->

## 用例场景
<!-- 3-5 个典型使用场景 -->

## 核心 Skill
<!-- 表格：类型、名称、来源链接 -->

## 评分
<!-- 四维度评分表 + 综合评分 -->

## 效果对比 / 技术特点
<!-- 根据 Skill 类型选择：
     - 效果对比：适合有明显视觉差异的 Skill（如 frontend-design）
     - 技术特点：适合功能型 Skill（如 xlsx）
-->

## 横向评测
<!-- 与竞品对比，包含测试任务、结果对比表 -->

## 使用示例 / 使用建议
<!-- 具体使用方法或最佳实践 -->

## 适用场景
<!-- 表格：场景、推荐程度 -->

## 局限
<!-- 3-5 条局限说明 -->

## 可选搭配
<!-- 可选，相关的 Skill/Hook/Agent -->
```

## 评分维度

| 维度 | 说明 | 评分标准 |
|------|------|---------|
| 效果 | 是否解决问题（含稳定性） | 5=完美 4=偶有瑕疵 3=需人工修正 |
| 复现成本 | 安装/配置复杂度 | 5=开箱即用 4=简单配置 3=需装依赖 |
| 通用性 | 不定制能否满足需求 | 5=拿来即用 4=稍作调整 3=需要定制 |
| 兼容性 | 客户端支持情况 | 5=全支持 4=2-3个 3=仅Claude Code |

## 写作规范

1. **语言风格**：客观陈述，避免口语化表达（如「太惊艳了」「没啥好说的」）
2. **图片命名**：存放于 `img/{序号}/` 目录，如 `img/001/effect-comparison.jpg`
3. **评分说明**：每个维度需附简短说明
4. **对比公平**：横向评测使用相同 Prompt

## 文件命名

```
usecases/{序号}-{英文名}.md
```

示例：
- `001-frontend-design.md`
- `002-xlsx.md`
- `003-playwright-auth.md`
