# 设计封面

> 使用设计哲学框架生成海报、封面等静态视觉艺术作品

## 用例场景

- 书籍/专辑/作品集封面设计
- 活动海报、展览宣传物料
- 品牌视觉识别系统
- 概念艺术、抽象艺术作品
- 数据可视化艺术

## 技术特点

### 设计哲学驱动

canvas-design 的核心不是直接生成图像，而是先生成设计哲学文档（`.md`），再根据哲学创作视觉作品。这个两步走的流程：

1. 将模糊需求转化为具体的美学语言
2. 强制建立一致的美学框架后再执行
3. 哲学文档可复用、可迭代

### 程序化生成

Skill 使用 Python（matplotlib、PIL、reportlab 等）程序化生成图像，而非 AI 文生图。这种方式的优势：

- **精确控制**：每个像素、线条、形状的位置和颜色都可精确控制
- **可重复性**：同样的代码每次运行结果一致
- **几何优势**：非常适合几何图形、图案、抽象艺术
- **专业字体**：Skill 内置 80+ TTF 字体文件可直接使用

### 局限性

程序化生成的固有局限：

- 缺乏自然感和有机感，难以创作手绘质感
- 无法生成真实的人物、动物、风景
- 作品容易显得"程序化"、"几何化"

## 使用建议

### 核心判断标准

如果需求是"90% 视觉设计 + 10% 必要文字"，并且追求艺术性和设计感，适合使用此 Skill。

### 推荐 Prompt 示例

**极简海报**：
```
/canvas-design 创建一个极简主义海报，主题是"寂静"，
使用大量负空间和单一几何形状，文字最少
```

**艺术展览海报**：
```
/canvas-design 为一个现代艺术展览设计入口海报，
展览名称："形式的记忆"，要求：
- 使用 Geometric Silence 哲学
- 黑白灰三色
- 文字不超过 20 个字
- 输出 PDF 格式
```

## 局限

1. **通用性较差**：需要特定的抽象/几何类设计需求才能发挥优势
2. **不支持具象内容**：无法生成"小狗"、"树懒"等真实物体
3. **学习曲线**：需要理解设计哲学框架才能写出有效的 Prompt
4. **仅支持静态输出**：输出格式限于 PDF 和 PNG，不支持动态效果
5. **风格单一**：产出偏向几何化、程序化的矢量风格
