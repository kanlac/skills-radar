# 评测标准

本仓库采用四维度评分体系，每个维度 0-5 分。

## 评分维度

### 1. 效果（Effectiveness）

**是否真的解决问题，包含稳定性考量**

| 分数 | 标准 |
|------|------|
| 5 | 完美解决，效果超出预期，从未失败 |
| 4 | 基本解决，偶有小瑕疵或需要重试 |
| 3 | 勉强可用，需要人工修正或经常失败 |
| 2 | 问题较多，成功率较低 |
| 1 | 几乎无法完成任务 |
| 0 | 完全无法使用 |

### 2. 复现成本（Setup Cost）

**安装/配置/依赖的复杂度**

| 分数 | 标准 |
|------|------|
| 5 | 开箱即用，无需配置 |
| 4 | 简单配置，几分钟搞定 |
| 3 | 需要安装依赖，但有清晰文档 |
| 2 | 配置复杂，可能遇到环境问题 |
| 1 | 需要大量手动配置和调试 |
| 0 | 几乎无法正常安装 |

### 3. 通用性（Generality）

**不需要定制化就能满足需求的程度**

| 分数 | 标准 |
|------|------|
| 5 | 拿来即用，适合绝大多数场景 |
| 4 | 稍作调整即可满足大部分需求 |
| 3 | 需要根据行业/团队规范做一定修改 |
| 2 | 内置规范较为特定，需要较多定制 |
| 1 | 仅适用于特定场景 |
| 0 | 几乎必须从头定制 |

### 4. 兼容性（Compatibility）

**客户端支持情况**

| 分数 | 标准 |
|------|------|
| 5 | 支持所有主流客户端 (Claude Code, OpenCode, Codex 等) |
| 4 | 支持 2-3 个主流客户端 |
| 3 | 仅支持 Claude Code，但可手动适配其他客户端 |
| 2 | 仅支持单一客户端，适配困难 |
| 1 | 有严重兼容性问题 |
| 0 | 仅限特定版本/环境 |

## 综合评分计算

综合评分 = 各维度分数的加权平均（默认权重相等）

## 横向评测说明

每篇评测文章会包含与竞品的横向对比：
- 同类 Skill 对比
- 专业 Agent 产品对比（如适用）
- 原生能力对比（不使用 Skill 的效果）
