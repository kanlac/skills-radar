# 评测标准

本仓库采用六维度评分体系，每个维度 0-5 分。

## 评分维度

### 1. 效果（Effectiveness）

**是否真的解决问题**

| 分数 | 标准 |
|------|------|
| 5 | 完美解决，效果超出预期 |
| 4 | 基本解决，偶有小瑕疵 |
| 3 | 勉强可用，需要人工修正 |
| 2 | 问题较多，实用价值有限 |
| 1 | 几乎无法完成任务 |
| 0 | 完全无法使用 |

### 2. 复现成本（Setup Cost）

**安装/配置/依赖的复杂度**

| 分数 | 标准 |
|------|------|
| 5 | 开箱即用，无需配置 |
| 4 | 简单配置，几分钟搞定 |
| 3 | 需要安装依赖，但有清晰文档 |
| 2 | 配置复杂，可能遇到环境问题 |
| 1 | 需要大量手动配置和调试 |
| 0 | 几乎无法正常安装 |

### 3. 稳定性（Stability）

**失败率、对环境敏感度**

| 分数 | 标准 |
|------|------|
| 5 | 多次测试从未失败 |
| 4 | 偶尔出错，但可自动恢复 |
| 3 | 有时需要重试或人工干预 |
| 2 | 经常失败，对环境敏感 |
| 1 | 成功率低于 50% |
| 0 | 几乎总是失败 |

### 4. 速度（Speed）

**执行耗时/交互成本**

| 分数 | 标准 |
|------|------|
| 5 | 秒级完成，交互极少 |
| 4 | 分钟级完成，交互简洁 |
| 3 | 需要数分钟，交互适中 |
| 2 | 耗时较长，交互繁琐 |
| 1 | 耗时过长，严重影响效率 |
| 0 | 无法在合理时间内完成 |

### 5. 可扩展性（Extensibility）

**是否容易改造成自己的工作流**

| 分数 | 标准 |
|------|------|
| 5 | 高度模块化，易于定制 |
| 4 | 提供扩展点，改造成本低 |
| 3 | 可以改造，但需要一定工作量 |
| 2 | 改造困难，耦合度高 |
| 1 | 几乎无法定制 |
| 0 | 完全封闭，无法扩展 |

### 6. 兼容性（Compatibility）

**客户端支持情况**

| 分数 | 标准 |
|------|------|
| 5 | 支持所有主流客户端 (Claude Code, OpenCode, Codex 等) |
| 4 | 支持 2-3 个主流客户端 |
| 3 | 仅支持 Claude Code，但可手动适配其他客户端 |
| 2 | 仅支持单一客户端，适配困难 |
| 1 | 有严重兼容性问题 |
| 0 | 仅限特定版本/环境 |

## 综合评分计算

综合评分 = 各维度分数的加权平均

默认权重相等（各 1/6），但具体用例可能会根据场景调整权重说明。

## 横向评测说明

每篇评测文章会包含与竞品的横向对比：
- 同类 Skill 对比
- 专业 Agent 产品对比（如适用）
- 原生能力对比（不使用 Skill 的效果）
