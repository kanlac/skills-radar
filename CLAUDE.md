# Skills Radar 项目指南

## 项目定位

精选 Claude Skills 评测仓库。以**用例（Use Case）**为单位撰写评测，而非 Skill 本身。

核心价值：告诉用户这个 Skill 怎么用、好不好用、替代方案有哪些。

## 文件结构

```
skills-radar-claude/
├── README.md          # 主页：定位、评测标准、文章索引、安装指南
├── RATING.md          # 四维度评分标准详解
├── TEMPLATE.md        # 评测文章撰写模板
├── CLAUDE.md          # 本文件，项目指南
└── usecases/
    ├── img/{序号}/    # 评测配图，按序号分目录
    └── {序号}-{英文名}.md  # 评测文章
```

## 评测标准（四维度）

| 维度 | 说明 |
|------|------|
| 效果 | 是否解决问题，含稳定性考量（瑕疵要减分） |
| 复现成本 | 安装/配置/依赖的复杂度 |
| 通用性 | 不需要定制化就能满足需求的程度 |
| 兼容性 | Claude Code / OpenCode / Codex 等客户端支持 |

综合评分 = 四个维度的平均分

## 写作规范

1. **语言风格**：客观陈述，避免口语化
2. **结构**：按 TEMPLATE.md 的目录结构撰写
3. **配图**：关键效果对比图必须包含，存放于 `usecases/img/{序号}/`
4. **评分说明**：每个维度需附简短理由
5. **横向对比**：使用相同 Prompt 测试，确保公平

## 新增评测流程

1. 确定序号：查看 `usecases/` 目录，取下一个序号
2. 创建文件：`usecases/{序号}-{英文名}.md`
3. 创建图片目录：`usecases/img/{序号}/`
4. 按 TEMPLATE.md 结构撰写内容
5. 更新 README.md 的评测文章索引表
6. 如有公众号原文，添加链接到文章开头

## 关键文件说明

- **README.md**：面向用户的主入口，包含安装指南（Claude Code / OpenCode / Codex）
- **RATING.md**：评分标准的详细定义
- **TEMPLATE.md**：新评测文章的目录结构模板
